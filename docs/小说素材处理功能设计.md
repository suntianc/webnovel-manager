# 小说素材处理功能设计

**版本:** 0.1
**日期:** 2026-04-25
**范围:** 手动上传小说文件，解析小说内容，抽取/整理素材，并接入现有素材库。

## 1. 目标

当前项目不再集成第三方小说搜索和下载能力，改为面向本地写作资料管理的“小说素材处理”工作流：

1. 用户手动上传小说文件。
2. 系统解析文件内容，识别章节结构。
3. 用户选择处理方式，生成候选素材。
4. 候选素材经确认后写入现有 `materials` 表。
5. 已上传小说、处理任务、处理结果可追踪、可复用、可删除。

核心原则：上传原文是资料源，素材是可复用的写作资产，两者需要分层保存。

## 2. 用户工作流

### 2.1 上传小说

用户上传 `.txt` 文件，后续可扩展 `.epub`、`.docx`、`.md`。

上传后系统记录：

- 小说标题
- 作者，可选
- 文件名
- 文件大小
- 编码识别结果
- 文件存储路径
- 章节数量
- 解析状态

### 2.2 解析小说

系统读取小说文本并拆分章节：

- 自动识别常见章节标题：`第1章`、`第一章`、`Chapter 1`、`卷一` 等。
- 保留章节序号、标题、正文、字数。
- 如果识别不到章节，则按固定字数切块，标记为“自动分段”。

### 2.3 处理小说

处理是一个任务，用户可以选择处理范围和处理类型。

处理范围：

- 全书
- 指定章节区间
- 单章
- 用户选中的文本片段

处理类型：

- 人物素材：角色名、身份、关系、性格、能力、关键事件。
- 世界观素材：势力、地名、制度、修炼体系、道具、规则。
- 情节素材：冲突、伏笔、反转、爽点、阶段目标。
- 文风素材：叙述方式、对话风格、描写片段、节奏样本。
- 金句/片段：可直接复用或参考的精彩表达。
- 自定义素材：用户手动框选并保存。

### 2.4 审核入库

处理结果先进入“候选素材”列表，不直接污染素材库。

用户可以：

- 编辑标题、摘要、正文。
- 调整分类、子分类、标签、评分。
- 合并重复候选。
- 忽略无价值候选。
- 一键或批量导入 `materials`。

导入后的 `materials.source_type` 建议设为 `小说上传`，`source_url` 可保存内部来源标识，例如 `novel://{novel_id}/chapter/{chapter_id}`。

## 3. 功能模块

### 3.1 小说源管理

接口能力：

- 上传小说文件
- 查看已上传小说列表
- 查看小说详情和章节列表
- 删除小说源
- 重新解析小说

### 3.2 章节管理

接口能力：

- 分页查看章节
- 查看单章正文
- 搜索章节内容
- 手动修正章节标题
- 合并/拆分章节，后续增强

### 3.3 素材处理任务

接口能力：

- 创建处理任务
- 查看任务列表
- 查看任务状态和进度
- 取消任务
- 查看任务日志

任务状态：

- `pending`: 等待处理
- `running`: 处理中
- `completed`: 已完成
- `failed`: 失败
- `canceled`: 已取消

### 3.4 候选素材

接口能力：

- 查看候选素材列表
- 编辑候选素材
- 忽略候选素材
- 导入单条候选到素材库
- 批量导入候选到素材库

候选状态：

- `draft`: 待审核
- `accepted`: 已导入
- `ignored`: 已忽略

## 4. 数据模型设计

### 4.1 novel_sources

保存上传的小说文件元数据。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| title | TEXT | 小说标题 |
| author | TEXT | 作者 |
| original_filename | TEXT | 原始文件名 |
| stored_path | TEXT | 本地存储路径 |
| file_size | INTEGER | 文件大小 |
| file_hash | TEXT | 文件 hash，用于去重 |
| encoding | TEXT | 文本编码 |
| status | TEXT | uploaded/parsed/failed |
| chapter_count | INTEGER | 章节数 |
| error_message | TEXT | 解析错误 |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

### 4.2 novel_chapters

保存解析后的章节。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| novel_id | INTEGER | 小说 ID |
| chapter_index | INTEGER | 章节序号 |
| title | TEXT | 章节标题 |
| content | TEXT | 章节正文 |
| word_count | INTEGER | 字数 |
| start_offset | INTEGER | 原文起始位置 |
| end_offset | INTEGER | 原文结束位置 |
| created_at | TEXT | 创建时间 |

建议对 `title`、`content` 建立 FTS5 表，便于章节内搜索。

### 4.3 novel_processing_jobs

保存处理任务。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| novel_id | INTEGER | 小说 ID |
| job_type | TEXT | 处理类型 |
| scope_type | TEXT | all/chapter_range/chapter/selection |
| scope_payload | TEXT | JSON，保存章节范围或选区 |
| status | TEXT | pending/running/completed/failed/canceled |
| progress | INTEGER | 0-100 |
| total_units | INTEGER | 总处理单元 |
| processed_units | INTEGER | 已处理单元 |
| error_message | TEXT | 错误信息 |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |
| completed_at | TEXT | 完成时间 |

### 4.4 material_candidates

保存待审核素材。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| novel_id | INTEGER | 小说 ID |
| chapter_id | INTEGER | 来源章节 ID |
| job_id | INTEGER | 来源任务 ID |
| title | TEXT | 候选标题 |
| content | TEXT | 候选正文 |
| summary | TEXT | 摘要 |
| category | TEXT | 一级分类 |
| subcategory | TEXT | 二级分类 |
| tags | TEXT | JSON 标签数组 |
| value_score | INTEGER | 0-5 |
| confidence | REAL | 自动处理置信度 |
| status | TEXT | draft/accepted/ignored |
| material_id | INTEGER | 导入后的素材 ID |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

## 6. 处理策略

### 6.1 第一阶段：规则处理

先实现不依赖外部 AI 的规则处理，保证系统可用：

- 章节拆分
- 人名/地名高频词初筛
- 含有设定关键词的段落提取
- 含有冲突关键词的段落提取
- 长段精彩片段保存
- 用户手动框选保存

适合关键词：

- 人物：他、她、少年、少女、长老、宗主、师父、弟子、敌人、主角姓名。
- 世界观：宗门、王朝、帝国、学院、秘境、灵气、境界、功法、法宝。
- 情节：危机、反转、伏笔、背叛、突破、追杀、交易、试炼。

### 6.2 第二阶段：AI 辅助处理

后续可在 Service 层增加 AI 处理器：

- 按章节摘要
- 抽取人物卡
- 抽取世界观卡
- 识别伏笔和爽点
- 给候选素材自动分类和打标签

AI 输出仍写入 `material_candidates`，由用户审核后再入库。

## 8. 前端页面设计

建议新增 3 个主页面：

### 8.1 小说库

展示已上传小说：

- 标题、作者、章节数、文件大小、解析状态、更新时间
- 上传按钮
- 重新解析、删除
- 点击进入小说详情

### 8.2 小说详情

左侧章节列表，右侧章节正文。

能力：

- 搜索章节
- 选择章节范围
- 创建处理任务
- 框选文本保存为素材候选

### 8.3 候选素材审核

类似工作台：

- 左侧筛选：小说、任务、分类、状态
- 中间候选素材列表
- 右侧编辑面板
- 支持导入、忽略、批量导入

## 9. 补充功能建议

优先级从高到低：

1. 文件去重：根据 hash 避免重复上传。
2. 编码识别：优先支持 UTF-8、GBK、GB18030。
3. 章节搜索：支持从原文中快速定位素材来源。
4. 来源回溯：素材详情能跳回原小说章节。
5. 批量操作：批量导入、批量忽略、批量打标签。
6. 处理任务重跑：调整规则后可重新生成候选素材。
7. 素材去重：候选入库前与现有素材做标题/正文相似度检查。
8. 导出：按小说或分类导出素材为 Markdown/JSON。

## 10. MVP 范围

第一版建议只做这些：

1. 上传 `.txt` 小说。
2. 自动解析章节。
3. 查看小说列表、章节列表、章节正文。
4. 手动选中文本保存为候选素材。
5. 规则提取“设定片段/人物片段/情节片段”候选素材。
6. 审核候选素材并导入现有素材库。

暂不做：

- 第三方下载
- 第三方搜索
- 实时爬取进度
- 多用户权限
- 复杂 AI 流水线

这个 MVP 和当前项目的数据结构最贴合，能最快把“小说原文 → 可复用素材”的闭环跑起来。
